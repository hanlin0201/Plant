# F2 聊天面板 · 交互说明

> 给开发看的功能描述，不是代码。
> 可运行 Demo 见同目录 `F2-chat-panel-demo.html`（浏览器直接打开）

## 设计方案：D（A + B 混合）

### 核心思路

两种交互深度，一个入口：
- **浅层互动（模式 A）**：点击植物 → 气泡 + 输入条，快速说一句话
- **深层对话（模式 B）**：点「展开」 → 侧边聊天面板，连续对话

用户不需要做选择，交互自然升级。

---

## 交互流程

### 1. 默认状态

植物在桌面右下角，头顶一个气泡，显示植物的主动发言。
气泡内容由传感器数据 + Prompt 生成。

### 2. 点击植物 → 出现输入条（模式 A）

- 植物下方弹出一个小输入条（input + 发送按钮 + 展开按钮）
- 用户打字，按回车发送
- 植物的回复显示在气泡里（替换原有内容）
- 适合快速互动：一问一答，然后继续工作

### 3. 点「展开」→ 聊天面板（模式 B）

- 输入条消失，植物旁边弹出一个 280×400 的聊天面板
- 面板顶部有「收起」按钮
- 历史对话（模式 A 里说过的话）会带入面板
- 支持连续多轮对话，带滚动
- 适合深聊：心情不好时、想跟植物多说几句时

### 4. 收起面板 → 回到输入条（模式 A）

- 点「收起」或按 Esc，面板关闭
- 回到输入条状态，气泡显示最后一句植物的话

### 5. 点击其他区域 → 全部收起

- 输入条和面板都收起
- 回到默认状态（只有植物 + 气泡）

---

## 气泡 vs 面板 的关系

| | 气泡 | 面板 |
|---|---|---|
| 显示位置 | 植物头顶 | 植物左侧 |
| 内容 | 最后一句植物的话 | 完整对话历史 |
| 输入方式 | 下方输入条 | 面板底部输入框 |
| 适合场景 | 快速一问一答 | 连续多轮对话 |
| 同时存在？ | 面板打开时气泡仍显示 | 是 |

**关键：面板里的对话也会同步更新气泡内容。** 这样即使面板被收起，用户还能看到植物最后说的话。

---

## 开发对接要点

1. **输入条和面板共享同一个对话历史数组**，切换时不丢数据
2. **植物回复走 AI API**：用户消息作为 user message，连同 sensor_data 和 chat_history 注入到 System Prompt
3. **回复前显示打字中状态**：气泡里显示 `...`，面板里也可以显示一个 typing indicator
4. **Esc 键**：面板模式按 Esc → 收起为输入条；输入条模式按 Esc → 全部收起
